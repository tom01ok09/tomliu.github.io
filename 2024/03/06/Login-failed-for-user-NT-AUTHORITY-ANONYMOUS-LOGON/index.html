Login failed for user 'NT AUTHORITY\ANONYMOUS LOGON'.

Posted on 2024-03-06 In SSIS

Issue

From Server1, using Windows Authentication to login to the SQL Server in Server2 which has the SSISDB to run the package and there is a connection using the Windows Authentication login back to the Server1, the service acount of SQL Server in server2 will need to enable deleation.

Fix

  1. Server2 service account need to trust this user for delegation to any service (Kerberos Only).
  2. Client machine restart or execute below command and restart client SSMS to clear cache.
# SSIS # Kerberos
[Azure Blob Source] Error: The remote server returned an error: (400) Bad Request.
Taking very long time from Azure SSIS-IR to download Blob storage files
