SHIR as a proxy for the Azure-SSIS IR : Timeout when reading from staging

Posted on 2024-03-03 Edited on 2024-03-04 In ADF , Azure-SSIS IR , SHIR

Issue

From the SSISDB report, we see below error:

Data Flow Task:Error: The Execute method on the task returned error code 0x80131505 (Timeout when reading from staging). The Execute method must succeed, and indicate the result using an “out” parameter.

Troubleshooting

Checking the log in the:

C:\ProgramData\SSISTelemetry\4633fb0b-0c8a-43c4-99fc-e240afd2c14a\4633fb0b-0c8a-43c4-99fc-e240afd2c14a

Found below error log:

Create Process failed. Exception=Microsoft.SqlServer.IntegrationServices.ScaleOut.Utilities.Credential.CreateProcessException: CreateProcessAsUser return false. cmd=’”C:\Program Files\Microsoft Integration Runtime\5.0\CommonExtensions\Microsoft\SSIS\140\Binn\ISServerExec.exe” /operation:execute /OperationMode:InWorker /Guid:6aa17f52-9be2-4921-9033-014b460892fb /OperationID:1’, workDirectory=’C:\ProgramData\SSISTelemetry\4633fb0b-0c8a-43c4-99fc-e240afd2c14a\4633fb0b-0c8a-43c4-99fc-e240afd2c14a\work’, domain=’TOM’, user=’sql_login_2’, errorCode=’1314’.

Fix

On-premises staging tasks on the SHIR require Windows authentication and on-premises staging tasks will be invoked with the SHIR service account.

SHIR service account:

Assign the Adjust memory quotas for a process
Assign the Replace a process level token policies
Restart the service

Windows Authentication account:

Assing the Log on as a service policy

Reference:
Error message: “Timeout when reading from staging”
Enable Windows authentication for on-premises tasks
